The file contains three columns: The first one contains the molecule label, the other two contain x,y-coordinates per molecule. The column names do not matter, but you need to keep their order if you work with your own data.

Scaling factor

In the Parameters section you can hand-tune the most important SSAM parameters.

SSAM-lite provides a preview function that processes small tissue sub-sets on the fly to enable parameter optimization in real time.
Click 'Use preview generator for parameter search' to enable it!

The parameters to optimize are:

  • Pixel width: The size of our discrete analysis and output area in pixels.
  • KDE kernel bandwidth: The 'smoothing factor' applied to the mRNA locations.
  • Expression threshold: Threshold for discarding areas with low expression signal.

We'll now have a look at each parameter in detail.

Close Next

The coordinates preview on the left lets you chose the subset of the tissue to calculate the preview on - simply click on the map to change it (but be patient as it might take a moment for the adjustment to be computed).

Below you can set the pixel width. This sets the size of the matrix the mRNAs are projected on. A larger value increases the resolution but more ressources will be required for the computations.

Close Next

Increasing the KDE kernel bandwidth (in μm) creates a smooting effect between the different mRNA spots, which integrates the mRNA signal. The center plot should help you optimize the value.

The rightmost plot shows the algorithm's final cell-type assignment.
Choose a bandwidth large enough to eliminate unwanted noise and at the same time retain the desired structural detail in the result.
Choose the threshold to remove unwanted, low-signal areas from the result.
